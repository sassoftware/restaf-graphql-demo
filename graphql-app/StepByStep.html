Adding your own web Application

Note: Instructions are for 1.6.0 version

Any web application can call the graphlql server. So it is not necessary to follow the instructions below. The instructions here assume that you want to extend this repository by duplicating wineProductionCas as a template

Step 1: Build the viewer in the src/viewers directory

  • Copy WineAppp.js to BeerAppp.js(Must be captilized as shown here)
  • Change the products and years arrays to your values.
  • Change all references to Wine(s), wine(s), to Beer(s) and wine(s) in the file
  • Save file
  • On the console issue this command "npm run build" to build the uiLibrary bundle

Step 2: Build the beer.html in public directory

  • Copy winecas.html to beer.html
  • Edit the call to uiLibrary(around line 72)
    •     uiLibrary.display('WineAppp', {graphqlQuery: 'wineProductionCas', host: host}, document.getElementById('display'));

    TO:

    uiLibrary.display('BeerAppp', {graphqlQuery: 'beerProduction', host: host}, document.getElementById('display'));
    The value of grapqhQuery (beerProduction) must match the root query you defined.

Step 3: Add the app to public/index.html

Add this line to the list as follows: 
<li> <a href="javascript:show('/beer.html')"  target="_blank" > Beer Production </a></li>

Step 4

Launch the apps as usual. You should be able to run the beer app now.
